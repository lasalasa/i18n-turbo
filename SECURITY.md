# Security Policy

## Supported Versions

We prioritize security updates for the latest stable release. Please ensure you are using the most recent version to stay protected.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0.0 | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a vulnerability in `i18n-turbo`, please report it responsibly.

### � Do NOT file a public issue
Please do not report security vulnerabilities through public GitHub issues, discussions, or pull requests.

### � How to Report
Email us details at **info@codecanvascollective.com**.

Please include:
- A description of the vulnerability.
- Steps to reproduce the issue.
- The version(s) of `i18n-turbo` affected.
- Any relevant code snippets or proof-of-concept.

### ⏳ Response Timeline
- **Acknowledgment**: Within 3 business days.
- **Assessment**: We will evaluate the severity and impact within 1 week.
- **Resolution**: A fix will be released as soon as possible, transparently communicated to you.

## Disclosure Policy

We follow a [Coordinated Vulnerability Disclosure](https://en.wikipedia.org/wiki/Coordinated_vulnerability_disclosure) process. We request that you do not publicly disclose the issue until we have had a reasonable opportunity to address it.

Thank you for helping keep the open source community safe! ❤️
